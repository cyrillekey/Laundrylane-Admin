## State Management
- **Server state**: `@tanstack/react-query` v5 â€” use auto-generated hooks from `queries/@tanstack/react-query.gen.ts`
- **Form state**: `@tanstack/react-form` v1 with `zod` validation
- **Client state**: Zustand with persist middleware for selected store

## API / Codegen
- `npm run codegen` reads OpenAPI spec from `http://localhost:4000/api/documentation/openapi.json`
- All API calls use auto-generated functions from `queries/` (never hand-write API clients)
- Pattern: `useQuery({ ...getXxxOptions({ query: { storeId } }), enabled: !!storeId })`
- Mutations: `useMutation({ ...postXxxMutation(), onSuccess: () => queryClient.invalidateQueries({ queryKey: getXxxQueryKey() }) })`
- **Every `useMutation` and `useQuery` must include an `onError` handler** that shows a toast with `(error as Error)?.message` or a contextual fallback string. Never leave errors unhandled.
- Errors: `throwOnError: true` is set globally in client config
- Eslint ignores `queries/**` â€” generated code is exempt from lint

## Data Tables (`@tanstack/react-table` v8)
Pattern â€” `container/{feature}/xxx-table.tsx`:
- `useReactTable` with `getCoreRowModel`, `getSortedRowModel`, `getPaginationRowModel`
- Columns: selection checkbox, data columns, actions column (dropdown with Edit/Delete)
- `SortHeader` component from `@hugeicons/core-free-icons` for sortable columns
- Skeleton loading state, empty state, pagination footer
- Bulk delete with AlertDialog confirmation
- Client-side search via `useMemo` filter on the page component

## Forms (`@tanstack/react-form`)
Pattern â€” `container/forms/{feature}/xxx-create-dialog.tsx`:
- `useForm` with `defaultValues` and `validators: { onChange: zodSchema }`
- `form.Field` render prop: `field.state.value`, `field.handleChange()`, `field.handleBlur()`, `field.state.meta.errors`
- `form.Subscribe` for `{ isSubmitting }` state
- `form.reset()` to clear after adding items
- Multi-item staging: fill draft form â†’ submit adds to a `items[]` array â†’ final submit sends batch

## Quirks & Conventions
- `@/` path alias maps to project root (not `src/`)
- `proxy.ts` at root is the Next.js middleware file (named `proxy.ts`, not `middleware.ts`)
- Files use `kebab-case.tsx` with `"use client"` directive
- Pages use route groups: `(auth)/`, `(app)/`, etc.
- Cloudinary images allowed in `next.config.ts` remotePatterns
- Sentry configured via `sentry.server.config.ts` + `sentry.edge.config.ts`
- Cloth types API has **no DELETE or UPDATE endpoints** â€” only GET and POST
- `NEXT_PUBLIC_BASE_URL` defaults to `https://laundry-lane-server.onrender.com` (set in local `.env` to `http://localhost:4000`)
- Firebase init reads from `firebase/config.json` (gitignored)
